# ChatWork-to-GoogleSheets2

Chatwork の Webhook を受け取り、タグ形式のメッセージを Google スプレッドシートに自動保存・管理する Google Apps Script プロジェクトです。

## 概要

このプロジェクトは [ChatWork-to-GoogleSheets](https://github.com/amada717/ChatWork-to-GoogleSheets) をベースに、**メッセージエンジニアリング**を重視して再設計されました。

### コンセプト

- **管理テンプレートによる「縛り」**: 自由度を制限し、必要最小限の構造化されたメッセージ形式を強制
- **作成者の負担軽減**: カテゴリーやタグ作成に時間を使わず、メッセージ本文の内容作成に集中
- **システマチックな処理**: バックエンドで自動的にメタデータを抽出・保存
- **段階的アプローチ**: WEB アプリ作成ではなく、まずシート保存と管理テンプレート設計に注力

## 設計思想の変遷

### 初期版（詳細テンプレート）
- カテゴリー、タグ、優先度など多数の選択項目
- [テンプレートメーカー](https://three-tree.xsrv.jp/demo/chatwork-template-generator/message_template2.html)で詳細な構造化
- ユーザーが事前に項目を選択・入力

### 現行版（**v2 タグ形式**）
- **投稿はすべて保存**（`<件名>` と `<目的>` の必須制約を廃止）
- `#会社#` `#期限日#` `#指示#` `#内容#` を必要なときだけ記載
- 本文は `###` 以降をそのまま保存
- メタデータ（投稿者・宛先・日時）はシステムが自動抽出

## 機能

### 🔄 Webhook 処理
- Chatwork からの POST リクエストを自動受信
- すべてのメッセージを処理（テンプレート必須条件なし）
- タグは存在する場合のみ抽出して保存

### 📊 データ管理
- **メインシート** (`room_v2_{room_id}`): 各メッセージの最新状態を保持
- **ログシート** (`log_room_v2_{room_id}`): すべての作成・更新履歴を記録
- **会社別シート** (`company_{会社名}`): `#会社#` があるメッセージを保存
- ルームごと、会社ごとに自動でシートを作成

### 🔍 重複・更新検知
- メッセージの重複投稿を検知して無視
- 編集されたメッセージは自動的に更新
- 変更履歴をログシートに記録

### 👥 メタデータ自動抽出
- **投稿者**: アカウント ID から名前を自動取得（Chatwork API 利用）
- **宛先**: `[To:123456]` 形式から複数の宛先を自動抽出
- **日時**: 送信・更新日時を自動記録
- **Chatwork リンク**: メッセージへの直接リンクを自動生成

### 🔒 排他制御
- `LockService` による並行実行の防止
- 最大 30 秒のロック待機

## 管理テンプレート形式

### 推奨形式

```
[To:123456]担当者名
#会社#サンプル株式会社
#期限日#2026-01-20
#指示#初回登録テスト
#内容#会社タグありの保存確認
###
本文内容をここに記載します。
```

### フィールド説明

| フィールド | 必須 | 説明 |
|----------|------|------|
| `[To:xxx]` | 任意 | 宛先のアカウント ID（複数可） |
| `#会社#` | 任意 | 会社名。値があれば `company_{会社名}` シートにも保存 |
| `#期限日#` | 任意 | 期限日（例: `2026-01-20`）。文字列として保存 |
| `#指示#` | 任意 | 指示内容。複数記載時は改行で連結 |
| `#内容#` | 任意 | 内容。複数記載時は改行で連結 |
| `###` | 推奨 | 本文の開始位置 |
| 本文 | 任意 | `###` 以降の自由記述内容 |

### 管理テンプレートの特徴

- ✅ **投稿制約なし**: テンプレート必須条件をなくし、すべて保存
- ✅ **必要時のみタグ指定**: `#会社#` などは必要なときだけ使える
- ✅ **メッセージ作成に集中**: 本文を `###` 以降にそのまま記述できる
- ✅ **自動メタデータ**: 投稿者・宛先・日時はシステムが自動で管理

## スプレッドシート構造

### メインシート (`room_v2_{room_id}`)

| カラム | 説明 | 自動/手動 |
|--------|------|----------|
| `メッセージID` | Chatwork メッセージへのリンク URL | 自動生成 |
| `投稿者` | 投稿者名 | 自動取得 |
| `宛先` | 宛先（カンマ区切り） | 自動抽出 |
| `会社` | `#会社#` の値 | 任意入力 |
| `期限日` | `#期限日#` の値 | 任意入力 |
| `指示` | `#指示#` の値（複数時は改行連結） | 任意入力 |
| `内容` | `#内容#` の値（複数時は改行連結） | 任意入力 |
| `本文` | `###` 以降の本文 | 任意入力 |
| `作成日時` | 作成日時 | 自動記録 |
| `更新日時` | 更新日時 | 自動記録 |

### ログシート (`log_room_v2_{room_id}`)

| カラム | 説明 |
|--------|------|
| `ログID` | ログの一意 ID（タイムスタンプ + message_id） |
| `メッセージID` | Chatwork メッセージへのリンク URL |
| `投稿者` | 投稿者名 |
| `宛先` | 宛先 |
| `会社` | 会社 |
| `期限日` | 期限日 |
| `指示` | 指示 |
| `内容` | 内容 |
| `本文` | 本文 |
| `操作種別` | アクション種別（`created` / `updated`） |
| `操作日時` | アクション日時 |

## セットアップ

### 1. Google スプレッドシートを作成

- [サンプルシート](https://docs.google.com/spreadsheets/d/1ebBi96EH86g9L26H-4J7Phj3VmSKXZCetlsB-T5O3Wg/edit?gid=130498908#gid=130498908)

### 2. Google Apps Script を設定

1. スプレッドシートから **拡張機能 > Apps Script** を開く
2. このリポジトリの `Code.js` の内容をコピー&ペースト
3. **プロジェクトの設定 > スクリプト プロパティ** から以下を追加:
   - プロパティ名: `CHATWORK_TOKEN`
   - 値: あなたの [Chatwork API トークン](https://www.chatwork.com/service/packages/chatwork/subpackages/api/apply_beta_business.php)

### 3. Web アプリとしてデプロイ

1. **デプロイ > 新しいデプロイ** を選択
2. 種類: **ウェブアプリ**
3. 次のユーザーとして実行: **自分**
4. アクセスできるユーザー: **全員**
5. デプロイ後、**ウェブアプリの URL** をコピー

### 4. Chatwork Webhook を設定

1. [Chatwork Webhook 管理画面](https://www.chatwork.com/service/packages/chatwork/subpackages/webhook/list.php) にアクセス
2. 対象のルームを選択
3. Webhook URL に上記の URL を設定
4. イベント: **メッセージが作成されたとき** を選択

## 使用例

### 1. Chatwork で管理テンプレート形式のメッセージを送信

```
[To:123456]山田太郎
#会社#サンプル株式会社
#期限日#2026-01-20
#指示#週次報告を作成
#内容#進捗状況の共有
###
今週の作業内容:
- タスクA完了
- タスクB進行中
```

### 2. 自動的にスプレッドシートに保存

- `room_v2_12345678` シートに最新データが記録される
- `log_room_v2_12345678` シートに作成ログが追加される
- `#会社#` がある場合は `company_{会社名}` シートにも記録される
- 投稿者・宛先・日時は自動で取得・記録される

### 3. メッセージを編集すると自動更新

- メインシートのデータが更新される
- ログシートに更新履歴が追加される（変更前のデータは残る）

## データフロー

```
Chatwork Webhook → doPost → タグ解析（任意）
                           ↓
                    既存メッセージチェック
            ├─新規 → room_v2 に追加 → ログに記録
            │        └─#会社#あり → company_シートにも追加
                    └─既存 → 内容比較
                           ├─変更あり → 更新 → ログに記録
                           └─変更なし → スキップ（重複防止）
```
