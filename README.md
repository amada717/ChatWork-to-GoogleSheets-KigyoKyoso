# ChatWork-to-GoogleSheets2

Chatwork の Webhook を受け取り、管理テンプレート形式のメッセージを Google スプレッドシートに自動保存・管理する Google Apps Script プロジェクトです。

## 概要

このプロジェクトは [ChatWork-to-GoogleSheets](https://github.com/amada717/ChatWork-to-GoogleSheets) をベースに、**メッセージエンジニアリング**を重視して再設計されました。

### コンセプト

- **管理テンプレートによる「縛り」**: 自由度を制限し、必要最小限の構造化されたメッセージ形式を強制
- **作成者の負担軽減**: カテゴリーやタグ作成に時間を使わず、メッセージ本文の内容作成に集中
- **システマチックな処理**: バックエンドで自動的にメタデータを抽出・保存
- **段階的アプローチ**: WEB アプリ作成ではなく、まずシート保存と管理テンプレート設計に注力

## 設計思想の変遷

### 初期版（詳細テンプレート）
- カテゴリー、タグ、優先度など多数の選択項目
- [テンプレートメーカー](https://three-tree.xsrv.jp/demo/chatwork-template-generator/message_template2.html)で詳細な構造化
- ユーザーが事前に項目を選択・入力

### 現行版（**管理テンプレート**）
- **必須項目を `<件名>` と `<目的>` の2つのみに削減**
- カテゴリー・タグは廃止し、**メッセージ本文の作成に集中**
- ユーザーの認知負荷を軽減し、入力の障壁を下げる
- メタデータ（投稿者・宛先・日時）はシステムが自動抽出

## 機能

### 🔄 Webhook 処理
- Chatwork からの POST リクエストを自動受信
- 管理テンプレート形式（`<件名>`と`<目的>`を含む）のメッセージのみを処理
- テンプレート形式でないメッセージは無視

### 📊 データ管理
- **メインシート** (`room_{room_id}`): 各メッセージの最新状態を保持
- **ログシート** (`log_room_{room_id}`): すべての作成・更新履歴を記録
- ルームごとに自動でシートを作成

### 🔍 重複・更新検知
- メッセージの重複投稿を検知して無視
- 編集されたメッセージは自動的に更新
- 変更履歴をログシートに記録

### 👥 メタデータ自動抽出
- **投稿者**: アカウント ID から名前を自動取得（Chatwork API 利用）
- **宛先**: `[To:123456]` 形式から複数の宛先を自動抽出
- **日時**: 送信・更新日時を自動記録
- **Chatwork リンク**: メッセージへの直接リンクを自動生成

### 🔒 排他制御
- `LockService` による並行実行の防止
- 最大 30 秒のロック待機

## 管理テンプレート形式

### 必須形式

```
[To:123456]担当者名
<件名>プロジェクトの進捗報告
<目的>週次報告による情報共有
[hr]
本文内容をここに記載します。
- 項目1
- 項目2
```

### フィールド説明

| フィールド | 必須 | 説明 |
|----------|------|------|
| `[To:xxx]` | 任意 | 宛先のアカウント ID（複数可） |
| `<件名>` | **必須** | メッセージの件名（全角英数字は半角変換） |
| `<目的>` | **必須** | メッセージの目的（全角英数字は半角変換） |
| `[hr]` | 推奨 | 本文の区切り線 |
| 本文 | 任意 | `[hr]` 以降の自由記述内容 |

### 管理テンプレートの特徴

- ✅ **最小限の構造**: 件名と目的だけを必須とし、本文は自由記述
- ✅ **入力の簡便性**: カテゴリー・タグなどの選択肢を排除
- ✅ **メッセージ作成に集中**: 項目選びではなく、内容作成に注力できる
- ✅ **自動メタデータ**: 投稿者・宛先・日時はシステムが自動で管理

## スプレッドシート構造

### メインシート (`room_{room_id}`)

| カラム | 説明 | 自動/手動 |
|--------|------|----------|
| `message_id` | Chatwork メッセージへのリンク URL | 自動生成 |
| `from_account` | 投稿者名 | 自動取得 |
| `to_accounts` | 宛先（カンマ区切り） | 自動抽出 |
| `subject` | 件名 | **手動入力** |
| `purpose` | 目的 | **手動入力** |
| `body` | 本文 | 手動入力 |
| `created_at` | 作成日時 | 自動記録 |
| `updated_at` | 更新日時 | 自動記録 |

### ログシート (`log_room_{room_id}`)

| カラム | 説明 |
|--------|------|
| `log_id` | ログの一意 ID（タイムスタンプ + message_id） |
| `message_id` | Chatwork メッセージへのリンク URL |
| `from_account` | 投稿者名 |
| `to_accounts` | 宛先 |
| `subject` | 件名 |
| `purpose` | 目的 |
| `body` | 本文 |
| `action_type` | アクション種別（`created` / `updated`） |
| `action_at` | アクション日時 |

## セットアップ

### 1. Google スプレッドシートを作成

- [サンプルシート](https://docs.google.com/spreadsheets/d/1ebBi96EH86g9L26H-4J7Phj3VmSKXZCetlsB-T5O3Wg/edit?gid=130498908#gid=130498908)

### 2. Google Apps Script を設定

1. スプレッドシートから **拡張機能 > Apps Script** を開く
2. このリポジトリの `Code.js` の内容をコピー&ペースト
3. **プロジェクトの設定 > スクリプト プロパティ** から以下を追加:
   - プロパティ名: `CHATWORK_TOKEN`
   - 値: あなたの [Chatwork API トークン](https://www.chatwork.com/service/packages/chatwork/subpackages/api/apply_beta_business.php)

### 3. Web アプリとしてデプロイ

1. **デプロイ > 新しいデプロイ** を選択
2. 種類: **ウェブアプリ**
3. 次のユーザーとして実行: **自分**
4. アクセスできるユーザー: **全員**
5. デプロイ後、**ウェブアプリの URL** をコピー

### 4. Chatwork Webhook を設定

1. [Chatwork Webhook 管理画面](https://www.chatwork.com/service/packages/chatwork/subpackages/webhook/list.php) にアクセス
2. 対象のルームを選択
3. Webhook URL に上記の URL を設定
4. イベント: **メッセージが作成されたとき** を選択

## 使用例

### 1. Chatwork で管理テンプレート形式のメッセージを送信

```
[To:123456]山田太郎
<件名>週次報告
<目的>進捗状況の共有
[hr]
今週の作業内容:
- タスクA完了
- タスクB進行中
```

### 2. 自動的にスプレッドシートに保存

- `room_12345678` シートに最新データが記録される
- `log_room_12345678` シートに作成ログが追加される
- 投稿者・宛先・日時は自動で取得・記録される

### 3. メッセージを編集すると自動更新

- メインシートのデータが更新される
- ログシートに更新履歴が追加される（変更前のデータは残る）

## データフロー

```
Chatwork Webhook → doPost → 管理テンプレート解析
                           ↓
                    既存メッセージチェック
                    ├─新規 → 追加 → ログに記録
                    └─既存 → 内容比較
                           ├─変更あり → 更新 → ログに記録
                           └─変更なし → スキップ（重複防止）
```
